# Sample Risk Register
### Pre-Launch Authentication Platform — Cloudflare Environment

| Risk ID | Risk Description | Category | Likelihood | Impact | Status | Controls Applied | Notes |
|---|---|---|---|---|---|---|---|
| R-001 | Automated bot traffic targeting the registration endpoint, creating fake accounts using gibberish names | Application Security / Data Integrity | High (recurring, distributed sources) | Medium (account integrity, resource consumption) | Mitigated | Registration endpoint rate limit; Cloudflare Managed Ruleset; OWASP Core Ruleset; custom rule targeting bot fingerprint | Blocked requests dropped from 5,400+ to single digits following mitigation |
| R-002 | Authentication emails (magic login links) delivered to spam folder instead of inbox during periods of increased email volume | Communication / Availability | Medium (triggered by external event, not constant) | High (directly affects user's ability to log in) | Resolved | Founder investigation linked the issue to DMARC configuration; resolved following remediation | Confirmed resolved through functional testing across multiple test accounts |
| R-003 | Discrepancy between the SSL/TLS edge certificate and the backup certificate expiration dates following automatic renewal | Infrastructure / Configuration | Low (isolated, non-recurring) | Medium (potential confusion or risk if unaddressed before launch) | Resolved | Raised with the founder for clarification; confirmed as dashboard synchronization delay; backup certificate updated accordingly | Demonstrates the importance of verifying automated renewal processes end-to-end |
| R-004 | Persistent low-level reconnaissance/probing activity (baseline-recon-block events) observed | Application Security / Reconnaissance | High (consistently present) | Low–Medium (no confirmed exploitation observed) | Monitored (Ongoing) | Cloudflare WAF baseline protections; continuous monitoring | No register-post or unauthorized access events observed during the monitoring period; recommend continued visibility post-launch |
| R-005 | Authorised WAF control exception exempting specific OAuth machine-to-machine endpoints from bot challenges, introduced to support third-party ecommerce integration | Integration Security / Access Control | Low (scoped, authorised, and intentional) | Medium (exempted endpoints could be targeted by automated traffic if not protected at the application layer) | Monitored (Ongoing) | Exemption scoped to named OAuth endpoint types only, reflecting principle of least privilege; founder confirmed change as authorised and intentional; broader WAF protections remain active across all other endpoints | Documented as a formal control exception consistent with change management principles under ISO 27001 (A.12.1.2) and NIST CSF (PR.IP-3); exemption scope expected to be reviewed and tightened as integrations develop |
